# Анализ обучения CartPole-агента

## Исходные данные

Прогон: **100 эпизодов**, команда `full` (build + node + render), seed 1234.

Метрики из лога (Ep 0000–0099):

| Метрика | Диапазон                    | Наблюдение                   |
| ------- | --------------------------- | ---------------------------- |
| Score   | 97–291                      | Флуктуации, без явного роста |
| Phase   | EXPLORATION                 | Всё время                    |
| Spike   | 0.000–0.003                 | Очень низкая активность      |
| Burst   | 0.00                        | Постоянно ноль               |
| OutBal  | -1.0 … 1.0                  | Случайные смещения           |
| Leak    | 747.7                       | Постоянно                    |
| Modes   | S:74k–219k, R:0, E:0, Rec:0 | Только stable                |

---

## 1. Конфигурация прогона

- **Adaptive Leak**: выключен (`adaptive_leak_mode=0` в `cmd_full`).
- **Дофамин**: `DOPAMINE_REWARD=10` при малой ошибке, `DOPAMINE_PULSE=0` при большой.
- **Autotuner**: включён, порог 700. Для перехода в DISTILLATION нужно SMA ≥ 350 (0.5 × target).
- **Night Phase**: `night_interval=10000`, всего ~15k шагов → ~1 Night Phase.

---

## 2. Оценка обучения

### 2.1. Score (длина эпизода)

- **Среднее**: ~150.
- **Тренд**: нет.
- **Сравнение**: Ep 0: 220, Ep 99: 97.

**Вывод**: за 100 эпизодов явного улучшения нет. Score ведёт себя как случайный.

### 2.2. Spike rate (0.000–0.003)

- **Смысл**: доля спайков за окно выборки.
- **Наблюдение**: ~0.002 → ~0.2% нейронов спайкуют в среднем.

**Вывод**: сеть активна слабо, но не «молчит». Это типично для разреженной спайковой сети.

### 2.3. Burst (0.00)

- **Смысл**: доля нейронов с burst_count ≥ 1.
- **Наблюдение**: всегда 0.

**Вывод**: burst-активность не возникает. Нейроны не переходят в режимы responsive/excited/recovery, поэтому adaptive leak (если бы был включён) не срабатывал бы.

### 2.4. Output balance (OutBal)

- **Смысл**: `left_spiking_count - right_spiking_count` в выходном слое.
- **Наблюдение**: от -1 до 1, без явного смещения.

**Вывод**: нет устойчивого выбора действия (left/right). Решение скорее случайное.

### 2.5. Effective leak (747.7)

- **Смысл**: средний leak по всем нейронам.
- **Вывод**: 747.7 ≈ 0.5×850 + 0.2×1275 + 0.3×223 — это базовый leak без модуляции.

### 2.6. Mode distribution (S:…, R:0, E:0, Rec:0)

- **Смысл**: доля нейронов в режимах stable/responsive/excited/recovery.
- **Наблюдение**: все нейроны в stable.

**Вывод**: без burst и без dopamine-модуляции adaptive leak не применяется. Все нейроны в stable.

**Примечание**: S растёт с длиной эпизода (score × batch_size × samples_per_step ≈ score × 20 × ~0.2 × samples). S ≈ 167040 для score 220 и 74240 для score 97 — это суммарное число «stable» за все samples эпизода.

---

## 3. Причины отсутствия обучения

### 3.1. Объём данных

- 100 эпизодов ≈ 15 000 шагов.
- R-STDP требует много повторений.
- Обычно для CartPole нужно десятки тысяч эпизодов.

### 3.2. Adaptive Leak выключен

- `cmd_full` использует `AdaptiveLeakConfig(adaptive_leak_mode=0)`.
- Нет модуляции leak по дофамину и burst.
- Сеть работает в режиме baseline.

### 3.3. Autotuner не переключается

- Порог для DISTILLATION: SMA ≥ 350.
- Фактические scores: 100–220.
- Переход в DISTILLATION не происходит.

### 3.4. Night Phase почти не участвует

- 1 Night Phase за прогон.
- Структурные изменения (pruning, sprouting) почти не применяются.

### 3.5. Дофамин

- Сигнал: 0–10 при error 0–1.
- R-STDP получает дофамин, но 100 эпизодов, скорее всего, недостаточно для заметного эффекта.

---

## 4. Рекомендации

### 4.1. Увеличить прогон

- Запускать хотя бы 1000–5000 эпизодов.
- Для оценки обучения — до 20000+.

### 4.2. Включить adaptive leak

- Запустить `benchmark` сценарии `dopamine_only` или `combined`.
- В `cmd_full` — передавать `AdaptiveLeakConfig` с `adaptive_leak_mode=1` и ненулевыми gain.

### 4.3. Сравнить с benchmark

- `python run.py benchmark --quick` для сравнения baseline и adaptive leak.
- `benchmark.py` уже использует `use_autotuner=False` и фиксированные параметры.

### 4.4. Проверить burst

- Burst = 0 может быть из-за:
  - низкой активности (spike_rate ~0.002),
  - порога burst_count или особенностей его счёта.
- Стоит проверить логику burst в `genesis-core` и `physics.rs`.

## 5. Краткий итог

| Симптом             | Вероятная причина                 |
| ------------------- | --------------------------------- |
| Score не растёт     | Мало эпизодов                     |
| Burst = 0           | Низкая активность или порог burst |
| Leak постоянный     | Adaptive leak выключен            |
| Phase = EXPLORATION | Scores не достигают порога        |
| OutBal случайный    | Недостаточно обучения             |
| R:0, E:0, Rec:0     | Нет burst, нет dopamine-модуляции |

**Общий вывод**: для данного прогона (100 эпизодов, baseline без adaptive leak) отсутствие обучения ожидаемо. Система работает, но объём данных и конфигурация не позволяют увидеть эффект R-STDP и адаптивной структуры.
